## Blog Agents — Copilot-Optimized

This repository includes a lightweight agent framework to orchestrate writing Medium-style blogs using the Copilot-optimized prompt process. The agents are intentionally simple, easily replaced with real web/search/model backends.

### Contents
- `blog_agents/` — agent implementations and small tools (mock web search, storage)
- `blog_runner.py` — simple CLI that runs the example pipeline
- `blog_data/` — artifacts generated by the pipeline (drafts, research JSON, hero prompt)
- `tests/` — pytest tests for basic agent behaviors
- `chatmodes.md` — definitions of interaction modes (intake, scan, outline, draft, edit)

### Quick start
1. Create a virtualenv and install pytest: `python -m venv .venv && . .venv/bin/activate && pip install pytest`
2. Run the example pipeline:
   - `python blog_runner.py "Why text editors matter" --anecdote "I once lost a patch in an editor crash"`
3. Run tests: `pytest -q`

### How agents map to the Copilot prompt
- IdeaExpander -> asks Socratic questions
- MarketResearcher -> web search summary
- ReferenceScout -> finds references
- OutlineReviewer -> quick outline QA
- MediumBlogger -> produces the draft (mocked)
- BlogEditor -> applies editing rules
- VisualPromptDesigner -> produces hero image prompt

## Copilot Chat Modes & Custom Agents

This repo includes templates and instructions to run the blog pipeline interactively from GitHub Copilot Chat (or by copy/pasting the included prompts).

- `copilot_modes/` contains JSON mode templates for typical roles: `Blog Writer`, `Researcher`, `Editor`. If your Copilot Chat version supports importing modes, you can import these files. If not, copy/paste the `instructions` text into a new Copilot Chat conversation.

- `chat_instructions/` contains ready-to-paste messages for each stage: intake, research, draft, and edit. Use these to drive the Copilot Chat conversation.

- `agents/custom_agent_template.md` describes how to add new agents (Python modules in `blog_agents/`) and the `chat_instructions` snippets to wire them into your chat flow.

If you'd like, I can also scaffold a small VS Code extension to make these modes available from a panel (not implemented by default; tell me if you want it).
