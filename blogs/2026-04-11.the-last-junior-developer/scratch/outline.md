# Outline: The Last Junior Developer

**Template**: 8. Then → Now → Next

## 1. How It Used to Work
*   **The Codebase Crawl**: Scenario 1 - First few weeks of a junior taking notes on a single feature, writing down architecture, to make a 10-line change.
*   **The Independent PoC**: Scenario 2 - Given a vague proof-of-concept just to learn the internal firm tooling, set up the environment, install libraries, run build pipelines, and open a PR.
*   **Pair Programming**: Scenario 3 - Two juniors blindly stumbling through problems together.

## 2. What Changed
*   The collapse of this onboarding phase via Agentic frameworks (LangChain, Claude Agent, Antigravity, CrewAI, AutoGen).
*   Code digestion in minutes, spec-kits eliminating environment set up. Epics of work getting done in hours with baked-in regression testing.
*   The raw productivity explosion for the senior developer acting as a shepherd.

## 3. Where We Are Now
*   **Desirable Difficulties (Malcolm Gladwell)**: Friction wasn't a bug in the old pipeline; it was a feature. Stumbling through the 10-line change is what built the cognitive map of the codebase.
*   When we remove the friction, we accelerate the task but kill the mastery.

## 4. What's Coming (The Judgment Deficit)
*   **Judgment vs. Generation (Adam Grant)**: If an AI is generating the code, the human must curate it with the mind of a scientist. 
*   But how do you evaluate an experiment (the AI's code) if you never went through the "grunt work" to build engineering intuition?
*   The threat of a long-term talent desert.

## 5. How to Prepare
*   The necessity to intentionally re-inject friction into learning paths.
*   Treating AI output as hypotheses to be verified, not solutions to be accepted.
