# Requirements Document

## Task Overview
Write one complete, publication-ready long-form blog post (3,000-4,500 words) about:
"What concepts from 1990s (and early-2000s) video games still linger today in society, as words/phrases and as the origins of modern concepts in systems design and AI."

## Audience
- Technology professionals (engineering, product, data, AI), mixed ages
- Assume many did NOT play 90s PC games
- Tone: warm, sharp, story-driven, not academic. No hype. No AI-cheerleading.

## Hard Style Rules
1. No horizontal divider lines
2. No em dash or en dash - use regular hyphen "-" only
3. Avoid contrast templates like "This isn't X. It's Y." and "X didn't do A. It did B."
4. Reduce use of the word "real"
5. Avoid semicolons
6. Expand acronyms on first use (then may use acronym)
7. Avoid AI-typical filler: "In today's world", "It's important to note", "This section will explore", "Let's dive in", "One thing is clear", "As we've seen"
8. Prefer concrete verbs and specific scenes over abstract commentary

## Scope Constraints
- Game cutoff: 1990s through early 2000s (roughly up to 2003)
- MUST include: 1993 Aladdin 2D platformer video game (not the movie)
- MUST include: Half-Life "wait-and-watch" emergent tactic (trigger a room, hide, let factions fight, then clean up) - connect to modern agent behavior (observation as an action)
- Reddit and Medium allowed as cultural sources, but primary/academic sources required for key claims

## Seed Games List (expand with others)
- Duke Nukem 3D, Doom, Wolfenstein 3D, Jazz Jackrabbit, Gods (Bitmap Brothers)
- Prince of Persia, Aladdin (1993 game), Half-Life
- Need for Speed (90s entries), Grand Theft Auto (top-down era), Pac-Man (ancestor)
- Quake, QuakeWorld, Unreal Tournament, Counter-Strike
- StarCraft, Age of Empires II, EverQuest, Ultima Online, Civilization II
- Diablo II, The Sims, Warcraft II/III (early 2000s)
- Mario Kart 64, Super Mario 64, Pokemon Red/Blue, Final Fantasy VII

## Research Requirements

### Concept Ledger (15-25 items)
Each row must include:
- Concept name (short, punchy)
- Layer: Language / Systems / AI
- Game(s) that illustrate it (at least one title)
- What the modern equivalent is (workplace, software, product, AI)
- Evidence type: academic / primary doc / cultural (reddit/medium) / common knowledge
- Source URLs (at least 1, ideally 2 per item; academic where possible)
- Notes on any uncertainty or competing origin claims

### Citations Pack
- Minimum 8 academic or peer-reviewed sources OR credible conference/workshop papers
- Topics: game AI, RL benchmarks, game studies, modding communities
- Include key primary sources (dev docs, GDC talks, platform docs, classic engine/mod docs)
- Cultural sources allowed only for memory, slang, community behavior, origin stories

### Fact-check pass
- Confirm non-obvious claims with sources
- For debated origins (smurfing, WAD expansion, etc.): use "often traced to" / "popularized by" rather than absolute claims

## Blog Structure

### H1: One strong title
- No colons if possible
- Clean and memorable

### Intro (180-250 words)
- Start with a modern scene (Slack, incident channel, code review, on-call)
- Use gaming language naturally

### Three H2 Sections (in order)
1. ## Language
2. ## Systems  
3. ## AI

### Under each H2: 5-9 H3 subsections
- Each H3 is one ledger item
- H3 must reference games in heading, like:
  `### "GG" as compressed sportsmanship (StarCraft, 1998; Quake, 1996)`
- Each H3 gets 1-2 paragraphs
- Vary length and rhythm
- Avoid repeating sentence openings/closings

### Conclusion (200-300 words)
- Tie Language -> Systems -> AI together
- Land a clear closing thought
- No cliches

## Content Guidance

### Language Section
- Show how terms helped strangers coordinate and shaped online norms
- Explain terms for younger readers
- Expand acronyms at first mention

### Systems Section
- Connect game design patterns to modern software:
  - Distribution (shareware)
  - Modding -> plugins
  - Engine/content separation
  - Observability via replays
  - Input standards (WASD)
  - Performance responsiveness
  - Ranking/metrics as product design

### AI Section
- Connect game mechanics to AI problems:
  - Partial observability (fog of war)
  - Navigation (Quake III style)
  - Replays as data
  - Multi-agent behavior
  - Scripted believability vs learned behavior
  - Doom -> ViZDoom
  - StarCraft research platforms
  - Half-Life "wait-and-watch" as observation/strategic non-action

## Output Requirements

### A) Concept Ledger + Citations Pack + 1-paragraph narrative arc summary

### B) Full blog draft in Markdown with H1/H2/H3 hierarchy

### C) Self-edit checklist report:
- List any repeated phrases removed
- Confirm no em/en dashes, no divider lines, no semicolons
- Confirm acronyms expanded on first use
- Flag any claims that are still "popularized by" rather than fully proven

## Quality Bar
- Must read like a human wrote it: specific, varied rhythm, minimal filler
- Avoid overclaiming origins. Prefer "popularized" and "made mainstream" to "invented" unless sourced
- Target 3,000-4,000 words. Max 4,500.
