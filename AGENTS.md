# AGENTS.md — Instructions for Amp

## Project Overview

This is a blog writing engine for Amp. The workflow is orchestrated by a specialized **Skill** that manages multiple agent phases.

## Authoritative Workflow: Blog Writing Skill

The primary workflow is defined in the **Blog Writing Skill**:
👉 [pipeline.md](file:///Users/kundansen/Documents/code/ink-and-iron/skills/blog_writer/pipeline.md)

### How to use
Reference the skill to start or resume a blog:
`@skills/blog_writer/pipeline.md`

### Stages
1. **Phase 0: Rubber Ducking** — Intellectual brainstorming and claim finding.
2. **Phase 1: Creative Brief** — Defining the "contract" for the piece.
3. **Phase 2: Research** — Parallel Maker-Checker deep dives.
4. **Phase 3: Skeleton** — Narrative architecture (v1, v2).
5. **Phase 4: Draft** — Prose generation and editing (Maker-Checker).
6. **Phase 5: Style Validation** — Vale linting and AI fallback checks.
7. **Phase 6: Assembly** — Final blog, hero image prompt, and social hook.

## Folder Structure

Each blog gets its own folder under `blogs/`, with a `scratch/` subfolder for internal working files.

```
blogs/<blog-slug>/
├── handoff.md         # Running state (authoritative)
├── brief.md           # Approved creative brief
├── <slug>-blog.md     # FINAL: the blog post
├── <slug>-hook.md     # FINAL: social sharing hook
├── <slug>-image.yml   # FINAL: hero prompt
└── scratch/           # Internal working files
    ├── research-*.md
    ├── skeleton-*.md
    ├── draft-v*.md
    └── style-check.md
```
